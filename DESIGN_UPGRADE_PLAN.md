# Board Game Room 디자인 고도화 계획서

작성일: 2026-07-02

## 1. 목적

현재 앱은 목재 테이블, 펠트, 컴팩트한 카드게임형 밀도, 게임별 기본 보드 UI를 갖춘 상태입니다. 다음 단계는 누락 없이 브랜드 이미지, 게임별 미니 보드, 조작 애니메이션, 상태 아이콘, 앱 설치 이미지를 체계적으로 보강하는 것입니다.

이 문서는 이후 작업 시 빠뜨리기 쉬운 이미지, 아이콘, 애니메이션, 검증 항목을 체크리스트로 관리하기 위한 실행 계획서입니다.

## 2. 현재 확인된 상태

### 이미 적용됨

- 전체 웹 UI는 목재/펠트 기반의 컴팩트한 테이블 인터페이스를 사용합니다.
- 게임 선택 목록에는 게임 종류별 아이콘, 인원수, 점수 방식, 선택 가능 상태가 표시됩니다.
- 설명 패널은 접힘 섹션으로 구성되어 있고, 세팅/턴 진행/구현 판정/승리 조건을 분리합니다.
- 게임별 실제 플레이 모듈 10개가 등록되어 있습니다.
- 비공개 정보 검증 스크립트와 전체 게임 플레이스루 QA가 존재합니다.

### 아직 부족함

- Android 아이콘과 스플래시가 기본 Capacitor 로고입니다.
- 웹 `index.html`에 favicon, web manifest, Apple touch icon, 공유용 OG 이미지가 없습니다.
- `BoardPreview` 컴포넌트는 존재하지만 등록된 게임 모듈이 있으면 fallback이라 실제 사용성이 낮습니다.
- 게임 선택/설명 화면에서 실제 게임판을 미리 보여주는 썸네일이 부족합니다.
- 애니메이션은 hover/transition 중심이고, 턴 변경/주사위/타일 공개/승리/타임아웃 같은 게임 행위 모션은 부족합니다.
- 게임별 조작 가능/불가능/비공개/시간초과/재접속 상태를 한눈에 보여주는 상태 아이콘 체계가 아직 완전하지 않습니다.

## 3. 디자인 방향

### 방향 A: Compact Walnut Table

- 현재 앱 흐름과 가장 잘 맞는 방향입니다.
- cardgames.io처럼 정보 밀도를 유지하되, 목재/펠트 질감과 실제 보드게임 오브젝트를 더 강하게 살립니다.
- 선택 방향입니다.

### 방향 B: Premium Board Box

- 각 게임을 보드게임 박스/컴포넌트처럼 보여주는 방향입니다.
- 시각적 매력은 높지만, 여러 명이 반복 플레이하는 앱에서는 화면 밀도가 떨어질 수 있습니다.

### 방향 C: Tournament Control Room

- 전적, 랭킹, 방 관리, 상태 확인을 중심으로 한 경기 운영형 UI입니다.
- 통계 화면에는 좋지만, 실제 보드게임 감성은 약해질 수 있습니다.

### 피해야 할 기본 패턴

- 단순 카드형 랜딩 페이지: 실제 게임 플레이 앱처럼 보이지 않습니다.
- 밝은 SaaS 대시보드 톤: 사용자가 요청한 목재/컴팩트 보드게임 느낌과 다릅니다.
- 장식용 이미지 남발: 조작 가능성이나 룰 이해에 도움을 주지 않으면 제외합니다.

## 4. 디자인 시스템 스냅샷

### 색상

- 주 배경: 짙은 월넛 목재
- 플레이 영역: 딥 그린 펠트
- 강조: 황동, 낡은 금속, 게임별 accent 색상
- 텍스트: 크림색 본문, 밝은 황동 헤딩, 녹색 성공 상태, 적갈색 위험 상태

### 질감

- 목재 grain: 앱 쉘, 패널, 방 코드, 게임 선택 행
- 펠트: 실제 플레이 보드, 선택 가능 영역, 현재 턴 강조
- 타일/카드/주사위 광택: 게임 오브젝트 전용
- 금속/황동 테두리: 아이콘 버튼, 상태 배지, 강조 포커스

### 모션

- 기본 시간: 120ms, 180ms, 260ms
- 보드 오브젝트 이동: transform 중심
- 공개/뒤집기: rotateY 또는 scale+opacity
- 상태 변화: 짧은 glow, pulse, shake
- `prefers-reduced-motion: reduce`에서는 비필수 모션 제거

### 시그니처 요소

다음 요소가 최소 5개 이상의 컴포넌트에 반복 적용되어야 합니다.

- 황동 아이콘 배지
- 목재 패널 프레임
- 펠트 선택 하이라이트
- 미니 보드/오브젝트 썸네일
- 짧고 절제된 조작 모션
