# UI/UX 개선 감사

## 1. Intent brief

- **사용자:** 규칙을 이미 알거나 빠르게 배워 친구와 바로 플레이하려는 1–4인 사용자
- **핵심 과업:** 방 생성 → 게임 탐색 → 시작 조건 확인 → 플레이 → 안전한 종료
- **감정 목표:** 공방의 물성은 유지하되, 다음 행동과 현재 상태를 망설이지 않고 이해한다.
- **제약:** 기존 게임 로직·소켓 계약·공방형 디자인을 유지하고 360px 터치, 키보드, 한글 가독성을 보장한다.

## 2. Direction proposals

1. **안내 강화형 공방:** 기존 시각 정체성 위에 검색, 준비 상태, 오류와 확인 절차를 명확히 둔다.
2. **경기 운영형:** 장식을 줄이고 점수·차례·행동만 높은 밀도로 재배치한다.
3. **터치 우선형:** 하단 고정 조작과 큰 제어를 중심으로 모바일을 우선한다.

선택은 1번이며, 플레이 헤더에는 2번의 상태 위계, 모바일에는 3번의 44px 조작 기준을 결합했다. 동일 크기 카드 대시보드, 파란 SaaS 상태색, 의미 없는 추가 애니메이션은 사용하지 않았다.

## 3. Design system snapshot

- 기존 Atelier 토큰의 Forest / Walnut / Brass / Ivory / Oxblood를 그대로 사용한다.
- 검색은 펠트 인셋, 준비 완료는 저채도 녹색, 오류는 옥스블러드 고정 알림으로 의미를 분리한다.
- 확인창은 황동 테두리와 2단 행동 위계(계속 플레이 / 위험 행동)를 사용한다.
- 모든 신규 입력·버튼은 최소 44px, 확인 행동은 48px이다.
- 기존 황동 모서리 시그니처는 헤더·검색·게임 박스·상태 칩·게임 셸·확인창에 유지된다.

## 4. Build plan and implemented map

- `GameShelfViewport`: 제목·원제·장르·요약·구성물 통합 검색, Escape 초기화, 결과 없음 복구, 결과 개수 안내
- `InteractiveGameLobby`: 시작 가능, 참가자 대기, 방장 대기, 선택 적용 중 상태 설명
- `PlayPanel`: 진행 중 로비 이동/방 나가기 확인창, Escape 취소, Tab 포커스 순환
- `PlayPanel`: 게임 보드 아래에 묻히던 오류를 상단 sticky alert로 이동하고 직접 닫기 제공
- 반응형 기준: 360 / 768 / 1280, 모바일 확인창은 React portal로 뷰포트에 고정

## 5. QA and critique report

- **Accessibility:** 검색 명시 라벨, 결과 `aria-live`, 확인창 `alertdialog`, 설명 연결, Escape 취소, Tab 순환 확인
- **Responsive:** 360 / 768 / 1280 가로 넘침 없음, 신규 입력·버튼 44px 이상
- **Motion:** 기존 reduced-motion 정책 유지, 신규 기능에 반복 애니메이션 없음
- **Performance:** 검색은 17개 로컬 배열 `useMemo` 필터, 추가 네트워크 요청과 이미지 없음
- **Consistency:** 기존 Atelier 토큰과 황동/펠트 깊이 전략 재사용
- **Swap test:** 검색·확인창도 공방형 인셋과 황동 위계를 사용해 일반 SaaS 입력으로 교환하면 정체성이 무너진다.
- **Squint test:** 게임 선택에서는 검색과 게임 박스, 플레이에서는 현재 상태와 보드, 이탈 시 확인창이 우선 보인다.
- **Signature test:** 헤더, 검색, 박스, 상태 칩, 게임 셸, 확인창 6개 영역에 반복된다.
- **Token test:** 신규 색·간격은 atelier 토큰과 4px 배수에 맞췄다.
- **Rebuild:** 1차 모바일 확인창이 변형된 부모를 기준으로 배치되는 실패를 발견해 `document.body` portal로 재구축했다.
