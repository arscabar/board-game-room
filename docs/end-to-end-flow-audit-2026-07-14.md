# 전체 사용자 흐름 재검증 · 2026-07-14

## Intent Brief

- 사용자: 방을 만들거나 공개 방을 찾고, 게임을 선택·설정·플레이한 뒤 안전하게 나가려는 1~4인 플레이어.
- 목표: 홈 → 방 → 게임 선택 → 플레이 → 일시정지/복귀 → 퇴장 흐름이 설명 없이도 이어지고, 오류와 파괴적 동작이 화면에서 명확히 보이게 한다.
- 정서: 기존의 짙은 펠트·목재·황동 보드게임 클럽 분위기를 유지하되, 장식보다 다음 행동과 현재 상태를 우선한다.
- 제약: 기존 게임 14개와 신규 게임 3개의 서버 권한·비공개 정보·타이머 규칙을 보존하고 360/768/1280px을 지원한다.
- 성공 조건: 전체 자동 QA 통과, 브라우저 실제 플레이 흐름 통과, 가로 넘침·의도치 않은 잘림·44px 미만 터치 대상 0건.

## 검토한 방향

1. **Flow-first club (채택)** — 기존 클럽 미감을 유지하고 상태, 주요 행동, 확인창, 오류 노출을 우선한다.
2. Screen-first showcase — 각 화면의 시각적 임팩트는 크지만 홈과 플레이 사이의 연속성이 약해져 제외했다.
3. Game-only minimal — 게임판 집중도는 높지만 방 관리와 복귀, 파괴적 동작의 맥락이 약해져 제외했다.

## Design System Snapshot

- 색: 야간 녹색 펠트, 어두운 월넛, 아이보리 본문, 황동 강조, 청록 연결 상태.
- 타이포: 디스플레이 서체는 테이블/게임명, 모노 계열은 방 코드와 시스템 상태에 제한한다.
- 형태: 작은 모서리 반경, 얇은 황동선, 입체 테이블은 유지하되 컨트롤은 평면적이고 명확하게 구분한다.
- 간격: 4/8/12/18px 계층, 모바일은 테이블 한 열과 44px 이상 조작 영역을 기본으로 한다.
- 동작: 선택은 포커스·pressed 상태와 상세 패널로 연결하고, 삭제/퇴장은 포커스가 갇힌 확인창을 거친다.

## Build Plan과 적용 결과

1. 홈과 공개 방: 투명한 3D 테이블의 히트 테스트를 보정해 카드 전체가 실제 버튼으로 동작하도록 했다. 진행 중 방은 “상세 보기”로 선택되고 비활성 “진행 중” 상태를 표시한다.
2. 방과 게임 선택: 고정 타이머 게임은 변경 가능한 셀렉트 대신 서버 규칙과 같은 고정 제한 시간을 표시한다.
3. 파괴적 동작: 네이티브 confirm을 키보드 포커스·Escape·Tab 순환을 지원하는 앱 확인창으로 교체했다.
4. 오류/안내: 방 단계와 플레이 단계 모두에서 서버 안내를 동일한 흐름 알림으로 노출한다.
5. 반응형: 420px 이하 테이블을 한 열로 바꾸고, 이름/게임명 줄바꿈과 상태 배지 원복, 입력·스킵 링크 터치 크기를 보정했다.

## QA와 Critique Report

### 기능 게이트

- 홈 → 방 생성 → 게임 선택 → 타이머 확인/변경 → 게임 시작 → 일시정지/재개 → 게임 중 퇴장 흐름 통과.
- 게임 선택 취소, 방 삭제 취소/확정, 진행 중 공개 방 상세 보기, 신규 3게임 실제 액션 흐름 통과.
- `qa:all` 통과: 프로덕션 빌드, 17개 카탈로그, 신규 3게임 규칙/소켓, 개인정보, 타임아웃, 17개 게임 시작 매트릭스와 완주 시뮬레이션.

### 반응형·접근성 게이트

- 360×900, 768×900, 1280×900: 문서 가로 넘침 0, 의도치 않은 내부 잘림 0, 44px 미만 조작 영역 0.
- 중복 ID, 레이블 없는 입력/버튼, 제목 단계 건너뜀, 대체 텍스트 누락 0.
- 브라우저 콘솔 error/warn 0.

### Critique loop

- **Squint test:** 황동 강조가 방 상태와 주요 행동에 집중되고 배경 장식이 흐름을 방해하지 않는다.
- **Hierarchy:** 현재 위치 → 화면 제목 → 테이블/게임 → 주요 행동 순서가 세 화면에서 유지된다.
- **Swap test:** 펠트 테이블, 목재 깊이, 황동선, 방 코드/상태의 제한적 모노 사용이 제품 고유성을 지킨다.
- **Signature moment:** 공개 테이블을 선택하면 입체 카드의 pressed 상태와 입장 카운터가 한 흐름으로 연결된다.
- **Token check:** 신규 보정은 기존 색·간격·모서리 변수를 재사용했고 임의의 새 시각 언어를 추가하지 않았다.

### 재빌드 기록

첫 검사에서 고정 타이머가 변경 가능한 컨트롤로 보이는 문제, 진행 중 방의 무반응, 360px 테이블 텍스트 잘림, 시각적으로 숨겨진 상태 배지의 불완전한 원복, 작은 입력/스킵 링크가 발견됐다. 각각 수정 후 동일 게이트를 다시 실행해 통과를 확인했다.
