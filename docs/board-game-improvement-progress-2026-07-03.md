# 보드게임 앱 개선 진행 보고서

작성일: 2026-07-03

## 완료한 작업

### P0. 공정성 및 게임 진행 안정화

- `GameModule`에 선택적 `applySystemAction` API를 추가했다.
- 서버의 일반 게임 액션과 시스템 액션이 같은 상태 반영 함수를 사용하도록 정리했다.
- 시스템 처리기가 없는 게임에서는 수동 공통 턴 종료를 차단하고, 타임아웃만 fallback으로 처리하게 했다.
- 다빈치 코드 플러스의 드로우/추측/결정 단계 타임아웃을 게임 룰에 맞게 처리했다.
- 요트 다이스에서 점수 기록 없는 수동 턴 종료를 차단하고, 타임아웃 시 가장 낮은 가능 점수칸을 자동 기록하게 했다.
- 블로커스에서 둘 수 있는 조각이 있으면 수동 패스를 차단하고, 타임아웃은 차례 넘김 페널티로 처리하게 했다.

### P1. 플레이 중 안내 및 모바일 조작성

- 공통 턴 종료 버튼에 게임별 제한 사유를 표시하고, 부적절한 수동 턴 종료를 비활성화했다.
- 쿼리도에 말 이동 후보 수, 선택한 벽 위치, 벽 배치 가능/불가 이유를 표시했다.
- 아발론 클래식에 선택한 구슬 수와 유효한 줄 선택 안내를 추가했다.
- 카왈레에 다음 분배 후보만 누를 수 있게 하고, 경로 선택 이유/상태 안내를 추가했다.
- 인쉬에 phase별 목표, 선택한 링, 이동 후보 수, 제거할 줄 수를 표시했다.
- 다빈치 코드 플러스에 시간 초과 자동 오답/팀 단서 안내를 추가했다.

### 비공개 정보 및 한 기기 플레이 위험

- 행맨 보드게임 setup 단계에서 본인 비밀 단어를 기본 숨김 처리했다.
- 한 기기 교대 플레이 시 `내 단어 확인`/`다시 숨기기`로 직접 노출을 제어하게 했다.

### QA

- `qa:timeouts` 스크립트를 추가했다.
- 다빈치 코드 플러스, 요트 다이스, 블로커스의 시스템 타임아웃/수동 턴 종료 특수 시나리오를 검증한다.

## 검증 결과

- `npm run build`: 통과
- `npm run qa:catalog`: 통과
- `npm run qa:privacy`: 통과
- `npm run qa:timeouts`: 통과
- `npm run qa:games`: 통과
- `npm run qa:max`: 통과

`qa:games`는 기본 4분 제한에서는 시간 초과되어, 10분 제한으로 재실행해 통과했다. 전체 시작 매트릭스와 10개 게임 자동 완주가 모두 성공했다.

`qa:max`는 각 게임별 최대 인원 기준 3판씩, 총 30판을 자동 플레이했다. 30판 모두 승자가 나오는 상태로 정상 종료했다.

## 아직 남은 작업

- 전체 10개 게임의 desktop/mobile 스크린샷 자동화는 아직 미완료다.
- 현재 프로젝트에는 Playwright/Puppeteer 같은 브라우저 자동화 의존성이 없다.
- 스크린샷 회귀 테스트를 정식으로 넣으려면 브라우저 자동화 의존성 추가 또는 별도 로컬 브라우저 실행 스크립트가 필요하다.
- 쿼리도, 블로커스, 인쉬의 실제 모바일 장기 플레이 감각은 추가 수동 QA가 필요하다.
- 전체 CSS 번들이 여전히 크다. 게임별 CSS 분리 또는 CSS-in-JS 정리가 다음 성능 개선 후보다.
