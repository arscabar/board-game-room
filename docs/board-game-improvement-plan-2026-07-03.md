# 보드게임 앱 개선 계획서

작성일: 2026-07-03

## 1. 목표

현재 앱은 10개 보드게임의 기본 플레이, 방 생성/입장, 인원 제한, 일부 비공개 정보 처리, 자동 완주 QA가 동작하는 상태다. 다음 개선 목표는 "친구들이 설명 없이도 각자 기기로 안정적으로 플레이할 수 있는 수준"으로 끌어올리는 것이다.

핵심 방향은 다음 네 가지다.

- 실제 보드게임 룰과 디지털 진행 상태가 어긋나지 않게 한다.
- 타임아웃, 턴 종료, 일시정지 같은 공통 기능이 게임별 룰을 깨지 않게 한다.
- 모바일에서도 보드/카드/말/벽/조각을 실수 없이 조작할 수 있게 한다.
- compact한 cardgame.io식 화면 밀도와 목재 보드게임 테이블 느낌을 함께 유지하되, 가독성을 해치지 않는다.

## 2. 현재 평가 요약

전체 평가: 친구끼리 실제 플레이 가능성 3.6 / 5

잘 되어 있는 부분:

- 10개 게임이 카탈로그에 등록되어 있다.
- 게임별 최소/최대 인원 제한이 있다.
- 주요 게임 자동 완주 QA가 통과한다.
- 구룡투, 고스트, 다빈치 코드 플러스, 행맨은 기본 비공개 정보 QA가 있다.
- 쿼리도, 블로커스, 행맨은 최근 UI 직관성 개선이 들어갔다.
- 게임 선택 화면, 설명 패널, 반응형 레이아웃, 목재 질감 테마가 적용되어 있다.

부족한 부분:

- 공통 강제 턴 넘김/타임아웃이 게임별 상태를 안전하게 정리하지 못할 수 있다.
- 다빈치 코드 플러스는 드로우 후 타임아웃 공정성 위험이 있다.
- 인쉬, 아발론, 카왈레는 초보자가 화면만 보고 가능한 행동을 이해하기 어렵다.
- 쿼리도, 블로커스, 아발론, 인쉬는 모바일에서 조작 정밀도가 부족하다.
- 행맨은 한 기기 교대 플레이 시 비밀 단어 노출 위험이 있다.
- 전체 10개 게임의 최신 UI 스크린샷 회귀 자동화가 부족하다.

## 3. 우선순위

### P0. 공정성 및 게임 진행 안정화

목표: 멀티플레이 중 시간 초과, 강제 턴 종료, 일시정지가 게임 룰을 깨지 않게 한다.

작업 항목:

1. 게임 모듈 API에 `handleTimeout` 또는 `applySystemAction` 계층을 추가한다.
2. 공통 서버의 `room:advance-turn`, `room:claim-timeout`, 자동 타임아웃이 게임별 안전 액션을 먼저 호출하게 한다.
3. 게임별로 턴 중간 상태를 정리하는 규칙을 만든다.
4. 강제 턴 넘김이 불가능한 phase에서는 사용자에게 이유를 반환한다.
5. 로그에 "누가, 어떤 이유로, 어떤 게임별 처리로 턴이 넘어갔는지"를 남긴다.

대상 파일 후보:

- `server/index.ts`
- `src/game-modules/types.ts`
- 각 게임의 `src/game-modules/*/index.tsx`

검증 기준:

- 타임아웃 후 active player, phase, private state가 서로 일치한다.
- 타임아웃 전후로 플레이어가 부당한 정보나 점수를 얻지 않는다.
- 일시정지 중에는 타임아웃이 진행되지 않는다.
- 방장이 시간 초과 전 임의로 상대 턴을 넘길 수 없다.

### P0. 다빈치 코드 플러스 드로우 후 타임아웃 수정

목표: 타일을 뽑은 뒤 아무 행동 없이 시간 초과되는 경우에도 공정성을 유지한다.

가능한 정책:

- 정책 A: 뽑은 타일을 보유하되 자기 타일 1개 공개 페널티를 준다.
- 정책 B: 뽑은 타일을 되돌리고 턴을 넘긴다.
- 정책 C: 자동 오답 처리 후 자기 타일 공개/보너스 카드 소비를 적용한다.

권장 정책:

- 실제 플레이 감각상 정책 C가 가장 명확하다. 다만 룰 문서와 화면 안내에 "추측 단계에서 시간 초과 시 오답 처리"를 명시해야 한다.

검증 기준:

- 드로우 직후 타임아웃 테스트가 추가된다.
- `drawnTileId`만 사라지고 타일 이득이 남는 상태가 없어야 한다.
- 2인/3인/4인, 팀전에서 모두 동일하게 처리된다.

### P1. 모바일 보드 조작성 개선

목표: 작은 화면에서도 오조작 없이 말을 놓고, 벽을 세우고, 조각을 배치할 수 있게 한다.

공통 작업:

1. 보드 영역에 확대/축소 또는 보드 맞춤 토글을 제공한다.
2. 모바일에서는 하단 고정 액션바를 사용한다.
3. 선택 후 바로 실행하지 않고 "미리보기 -> 확정" 흐름을 제공한다.
4. 잘못된 위치를 눌렀을 때 금지 이유를 즉시 표시한다.
5. 가로모드 권장 안내를 보드 상단의 작은 배너로 제공한다.

게임별 대상:

- 쿼리도: 벽 후보 위치, 방향, 경로 차단 불가 사유 표시
- 블로커스: 조각 선택/회전/뒤집기/배치 확정 단계 강화
- 아발론 클래식: 선택한 구슬 라인, 이동 방향, 밀기 가능 여부 표시
- 인쉬: 링 선택, 이동 경로, 뒤집힐 마커, 제거할 5목 표시

검증 기준:

- 390px 폭 화면에서 주요 버튼과 보드 셀이 겹치지 않는다.
- 한 손 조작 기준으로 최소 터치 영역이 확보된다.
- 확정 전 미리보기 상태를 취소할 수 있다.

### P1. 초보자용 실시간 행동 안내 강화

목표: 룰북을 열지 않아도 "지금 내가 뭘 해야 하는지"를 알게 한다.

공통 작업:

1. 각 게임 상단에 현재 목표, 가능한 행동, 금지된 행동 이유를 표시한다.
2. phase별 안내 문구를 게임 모듈이 제공하도록 정리한다.
3. 룰 설명 패널은 접이식으로 유지하고, 플레이 중 안내는 더 짧게 유지한다.
4. 액션 버튼은 현재 가능한 것만 강조하고, 불가능한 버튼은 이유를 tooltip 또는 작은 문구로 표시한다.

우선 대상:

- 인쉬: 배치, 이동, 5목 선택, 링 제거 단계 안내
- 아발론 클래식: 구슬 선택 개수, 이동 방향, 밀기 조건 안내
- 카왈레: 스택 분배 순서와 되돌아가기 금지 안내
- 다빈치 코드 플러스: 드로우, 추측, 계속 추측, 패스, 보너스 카드 처리 안내

검증 기준:

- 각 phase에서 최소 1개 이상의 명확한 다음 행동 문구가 보인다.
- 불가능한 행동을 했을 때 서버 에러만 뜨지 않고 화면 문맥에서 이유를 알 수 있다.

### P1. 비공개 정보 및 한 기기 플레이 모드 정리

목표: 각자 기기 플레이와 한 기기 교대 플레이에서 비밀 정보가 섞이지 않게 한다.

작업 항목:

1. 행맨에 "각자 기기 모드"와 "한 기기 교대 모드"를 구분한다.
2. 한 기기 교대 모드에서는 비밀 단어 입력/확인 화면을 별도 잠금 단계로 둔다.
3. 다빈치 코드 플러스의 팀원 힌트 `teamClue`가 실제 룰 의도인지 확정하고, 아니라면 제거하거나 옵션화한다.
4. 비공개 정보 QA에 viewer별 snapshot 비교를 더 촘촘하게 추가한다.

검증 기준:

- 상대 화면에서 내 비밀 단어/타일/정체가 노출되지 않는다.
- 한 기기 교대 모드에서 다음 플레이어가 이전 플레이어의 비밀 정보를 볼 수 없다.
- 다빈치 팀전 정보 공개 규칙이 UI와 룰 설명에 동일하게 적혀 있다.

### P2. 실물 보드게임 느낌 보강

목표: 실제 보드게임의 직관성을 더 살리되, 가독성과 compact UI를 유지한다.

작업 항목:

1. 구룡투: 숨겨 선택한 패가 동시에 공개되는 연출 강화
2. 다빈치 코드 플러스: 타일 랙, 흑백 타일 대비, 공개/비공개 상태 강화
3. 요트 다이스: 주사위 질감, 점수표 카테고리 비교 강조
4. 고스트: 선/악 유령 말의 촉감과 setup 슬롯 대응성 강화
5. 카왈레: 쌓인 돌의 높이와 분배 방향 표시 강화
6. 전체: 목재 배경 위 텍스트 대비, 버튼 대비, 상태 뱃지 대비 재검토

검증 기준:

- 배경 질감 때문에 글자가 흐려지지 않는다.
- 주요 상태 텍스트 대비가 WCAG AA 수준에 가깝게 유지된다.
- 실물 구성물의 역할을 처음 보는 사용자도 구분할 수 있다.

### P2. QA 및 회귀 테스트 자동화

목표: 새 게임 추가와 UI 수정 후에도 기존 게임이 깨지지 않게 한다.

작업 항목:

1. 전체 10개 게임의 desktop/mobile 스크린샷 캡처 스크립트를 정리한다.
2. 각 게임별 최대 인원 3판 자동 플레이 결과를 JSON/PNG로 저장한다.
3. 비공개 정보 QA를 게임별로 확대한다.
4. 특수 룰 시나리오 테스트를 추가한다.
5. UI 크기, 텍스트 겹침, 보드 overflow를 검사하는 Playwright 체크를 추가한다.

특수 시나리오:

- 다빈치: 드로우 직후 타임아웃, 정답 후 계속/패스 전 타임아웃, 4인 팀전 정보 노출
- 쿼리도: 4인전 연속 인접 말 점프/대각 우회, 벽으로 경로 완전 차단 시도
- 블로커스: 1칸 조각, 회전/뒤집기 후 모서리 접촉 배치, 놓을 수 없는 플레이어 패스
- 인쉬: 동시 5목, 6개 이상 연속 마커, 마커 소진, 링 제거 선택
- 행맨: 한 기기 교대 사용 시 비밀 단어 노출 방지
- 아발론: 3개가 2개를 밀어 보드 밖으로 제거하는 경계 상황

검증 기준:

- `npm run build` 통과
- `npm run qa:catalog` 통과
- `npm run qa:privacy` 통과
- `npm run qa:games` 통과
- 전체 게임 스크린샷 저장
- P0 특수 시나리오 테스트 통과

## 5. 실행 순서

### 1단계: 안정성 패치

1. 게임 모듈 timeout/pass API 설계
2. 서버의 공통 턴 넘김 연결 변경
3. 다빈치 드로우 후 타임아웃 처리
4. P0 테스트 작성
5. catalog/privacy/games QA 실행

완료 기준: 타임아웃이 공정성을 해치지 않는다.

### 2단계: 모바일 핵심 게임 조작성

1. 쿼리도 벽 배치 preview 개선
2. 블로커스 조각 미리보기/확정 개선
3. 아발론 선택 라인/방향 안내 개선
4. 인쉬 이동/5목 안내 개선
5. 390px/768px/1280px 스크린샷 저장

완료 기준: 작은 화면에서도 오조작 없이 플레이 가능하다.

### 3단계: 초보자 안내와 비공개 모드

1. phase별 행동 안내 공통 구조 정리
2. 인쉬/아발론/카왈레/다빈치 안내 강화
3. 행맨 한 기기 교대 모드 추가
4. 다빈치 팀 힌트 규칙 명시 또는 옵션화
5. viewer별 비공개 QA 강화

완료 기준: 처음 하는 사람이 화면만 보고 다음 행동을 이해할 수 있다.

### 4단계: 실물감 및 가시성 polish

1. 카드/타일/말/돌 질감 보강
2. 목재 테마 대비 재검토
3. 상태 뱃지와 버튼 색상 정리
4. 게임별 구성물의 역할이 잘 보이는지 스크린샷 리뷰

완료 기준: 보드게임 느낌은 강화되지만 글자와 조작성이 흐려지지 않는다.

### 5단계: 전체 회귀 자동화

1. 전체 게임 desktop/mobile 스크린샷 자동 캡처
2. 최대 인원 3판 자동 플레이 로그 저장
3. 주요 특수 룰 시나리오 추가
4. 결과를 `reports`에 JSON/PNG/MD로 저장

완료 기준: 이후 새 게임을 추가해도 회귀 테스트로 깨진 부분을 빨리 찾을 수 있다.

## 6. 완료 판정 기준

전체 개선 완료는 다음 조건을 모두 만족할 때로 본다.

- 모든 P0 이슈가 수정되어 공정성 문제가 없다.
- 10개 게임 모두 최대 인원 기준 자동 완주가 가능하다.
- 주요 비공개 정보가 viewer별로 보호된다.
- 모바일 390px 기준으로 핵심 게임판 조작이 가능하다.
- 각 게임마다 현재 목표/가능 행동/불가능 이유가 표시된다.
- 전체 게임의 최신 스크린샷과 QA 결과가 `reports`에 저장된다.
- git diff 기준 불필요한 대규모 리팩터링 없이 변경 범위가 설명 가능하다.
